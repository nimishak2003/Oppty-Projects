15. JavaScript Modules

Organize code using export and import (ES6 Modules)

What are Modules?

A module is a JavaScript file that contains code (variables, functions, classes) that can be reused in other files using export and import.

Modules help you write clean code by splitting big programs into small files.

Why Modules?

When projects grow, writing everything in one file becomes difficult. Modules help by:

✅ Reusability (use code in multiple places)
✅ Better folder structure
✅ Easy maintenance
✅ Avoid global variable conflicts

Named Export

You can export multiple values using named export.

// math.js
export const add = (a, b) => a + b;
export const sub = (a, b) => a - b;

// app.js
import { add, sub } from "./math.js";

console.log(add(5, 2)); // 7

Default Export

Default export is used when you want to export only one main thing from a file.

// greet.js
export default function greet(name) {
  return `Hello ${name}`;
}

// app.js
import greet from "./greet.js";

console.log(greet("Alex"));

Import Everything

You can import everything from a module using * as.

// math.js
export const add = (a, b) => a + b;
export const mul = (a, b) => a * b;

// app.js
import * as math from "./math.js";

console.log(math.add(2, 3)); // 5
console.log(math.mul(2, 3)); // 6

Important Note: type="module"

To use ES6 modules in the browser, your script tag should be:

<script type="module" src="app.js"></script>

Localhost Tip

Modules usually don’t work with direct file opening (file://). Run your files using Live Server or localhost.
