12. JavaScript Error Handling

Handle runtime errors using try, catch, finally and throw

What is Error Handling?

Error handling allows you to manage runtime errors in JavaScript without breaking your entire program.

In JavaScript, errors are handled using try, catch, finally, and throw. 

try {
  // code that may cause error
} catch (error) {
  // code to handle error
}

Try and Catch

The try block contains risky code. If an error occurs, the catch block handles it. 

try {
  let result = x + 10; // x is not defined
} catch (err) {
  console.log("Error occurred:", err.message);
}

The Finally Block

The finally block runs whether an error occurs or not. 

try {
  console.log("Executing...");
} catch (err) {
  console.log("Error");
} finally {
  console.log("Always runs");
}

Throwing Custom Errors

The throw statement allows you to create custom errors. 

function checkAge(age) {
  if (age < 18) {
    throw "You must be 18+";
  }
  return "Access Granted";
}
